# Content Negotiation

The same gateway URL can return three fundamentally different representations of a W4 page,
selected by the `Accept` header. This is the mechanism that makes a single resource useful
to browsers, AI agents, and parser pipelines simultaneously.

---

## The Three Representations

| Accept header | Content-Type returned | Audience |
|---|---|---|
| `application/w4ml+xml` | `application/w4ml+xml` | W4ML parsers, developer inspection |
| `application/w4+json` | `application/w4+json` | AI agents, tool discovery systems |
| `text/html` (or absent) | `text/html` | Browsers, human readers |

The gateway always responds with `Vary: Accept` so caches key correctly on this dimension.

---

## `application/w4ml+xml`

Returns the raw W4ML document exactly as it was loaded from disk (after path safety checks).
No transformation is applied.

```bash
curl -H "Accept: application/w4ml+xml" http://localhost:8090/
```

Use this to:
- Inspect the exact source document
- Feed into an external W4ML parser
- Debug parser issues

---

## `application/w4+json`

Returns the **compiled JSON representation** of the document, structured for agent consumption.
This is the primary format for AI agents discovering and invoking W4 services.

```bash
curl -H "Accept: application/w4+json" http://localhost:8090/ | jq .
```

The response structure includes:

```json
{
  "id": "https://example.com/page.w4",
  "title": "My Page",
  "sections": [...],
  "services": [
    {
      "id": "math.add",
      "name": "Add Numbers",
      "intent": "Adds two integers.",
      "kind": "tool",
      "input": { /* JSON Schema */ },
      "output": { /* JSON Schema */ },
      "effects": "none",
      "consent": "open",
      "x-w4": { /* W4-specific metadata */ }
    }
  ],
  "imports": [...],
  "peers": [...]
}
```

Each service is compiled to an **OpenAI-like tool JSON** format (with an `x-w4` extension
block for Web4-specific metadata), making the output directly usable with most AI tool-calling
frameworks.

---

## `text/html`

Returns an HTML rendering of the document, generated by a Tera template engine.

```bash
curl -H "Accept: text/html" http://localhost:8090/
# or simply open http://localhost:8090/ in a browser
```

**Template resolution order:**

1. If the document declares `<link rel="template" href="...">` in `<head>`, that template is
   loaded (local path or remote if `allow_remote` is enabled in config).
2. If no template is declared, the built-in default template is used.

**Load strategy in HTML rendering:**

| `load` value | HTML behaviour |
|---|---|
| `eager` | Rendered fully in the initial response |
| `lazy` | Rendered as a collapsed/folded element |
| `never` | Omitted from HTML output entirely |

The body order of sections and services is preserved as declared in the W4ML source.

See [Custom Templates](../guides/custom-templates.md) for how to write and associate templates.

## `w4fragment` — Fragment Selector

Any request (regardless of Accept type) may include a `w4fragment` query parameter to select
a specific sub-element of the document:

```
GET /?w4fragment=section:intro        → returns only the "intro" section
GET /?w4fragment=service:math.add     → returns only the "math.add" service
```

```bash
curl -H "Accept: application/w4+json" \
  "http://localhost:8090/?w4fragment=service:math.add"
```

**Selector syntax:** `type:id` where `type` is `section` or `service` and `id` is the element's
`id` attribute value.

**Error if not found:**

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "fragment 'service:unknown' not found",
    "retryable": false
  }
}
```

HTTP status: `404 Not Found`.

---

## Accept Header Priority

When the `Accept` header lists multiple types, the gateway uses first-match priority in the
order: `application/w4ml+xml` → `application/w4+json` → `text/html`. If none match or the
header is absent, the gateway defaults to `text/html`.

---

## `Vary` Header

Every document response includes:

```
Vary: Accept
```

This ensures HTTP proxies and CDNs cache the three representations separately.
